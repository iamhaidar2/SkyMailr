Quick Start

This Quick Start guide for SkyMailr walks through what you need to connect an app and send mail. Each area of the portal has a specific job; you can open the linked pages when you are ready.

Connect your app end-to-end

Typical order:

  1. Create a connected app (your product or environment) under Connected apps.
  2. Add an API key for that app on API keys (open the app, generate a key, copy it once).
  3. Configure a sender profile so messages have a valid From identity: Sender profiles.
  4. Create and approve email templates your API or workflows will use: Email templates.
  5. Optionally define workflows for lifecycle automation: Workflows.
  6. Invite members so teammates can help manage content or billing: Members.

The dashboard setup checklist reflects the same milestones in a compact checklist.

Sender profiles

Sender profiles tie a display name and mailbox to your connected app. The API references them when sending so recipients see a consistent From line that matches your verified domain.

Open sender profiles →

Email templates

Templates hold HTML and text for reusable messages. After you save a version, an owner or admin approves it for production; only approved templates can be used by the send API for that app.

Open email templates →

Workflows

Workflows let you automate multi-step mail (for example enrollments or follow-ups) without calling the send API for every step manually.

Open workflows →

Members

Members are people who can sign in to this SkyMailr account. Roles control who can approve templates, edit content, manage connected apps, invite others, or view usage.

Open members →
