Addresses SkyMailr will not deliver to, based on bounces, complaints, unsubscribes, or rules you add. Rows marked Global are set by the platform and cannot be removed here.

Why, when, and how

  • Why: Suppressions protect recipient trust and your sending reputation. After a hard bounce or spam complaint, continuing to send can hurt deliverability for everyone on the platform.
  • When: Use a manual block when someone asks to be removed from non-transactional mail but you still send them receipts, when you must stop a mistaken campaign to one address, or when debugging a specific recipient.
  • How: Choose whether the block applies to marketing / lifecycle only (default) or also to transactional / system mail—transactional blocks are powerful; use only when necessary. You can also manage blocks via the API (POST /api/v1/suppressions/).
