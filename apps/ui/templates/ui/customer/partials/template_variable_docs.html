{# Expects: template (EmailTemplate) #}

Use Jinja2 placeholders in subject and body, e.g. {% verbatim %}{{ user_name }}{% endverbatim %}. When your app sends this template via the API, pass a JSON context object whose keys match the variable names below (required keys must be non-empty strings).

Example: API request body

POST /api/v1/messages/send-template/ with Bearer token (tenant API key). 

{
  "template_key": "{{ template.key }}",
  "to_email": "user@example.com",
  "to_name": "Alex",
  "message_type": "transactional",
  "context": {
    "product_name": "My SaaS",
    "user_name": "Alex",
    "action_url": "https://app.example.com/verify?token=…"
  },
  "source_app": "my_backend"
}
